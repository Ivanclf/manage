# 高校学生活动全流程管理系统

## 作业要求原文

**高校学生活动全流程管理系统（难度：较难，基准分90）**

某高校想开发一款系统用于管理各类学生活动的发布、报名和签到，例如讲座、宣讲会、就业指导会等各类活动。为方便使用，该系统的正式用户仅为活动管理员，参与人不需要注册但可以报名和签到，系统应包含以下功能：
1. 活动的生命周期管理（活动管理员）
    1. 活动创建：填写活动的基本信息，如名称、类型（讲座、宣讲会、就业指导等）、活动简介、活动时间、地点、报名截止时间和人数上限等。（可选信息：可添加直播链接和附件）
    2. 活动查询：列表显示所有活动信息，包括活动状态（未发布/报名中/已截止/已结束）、发布时间、已报名人数等。可选：可根据多条件进行筛选（按类型/状态/时间范围）
    3. 活动发布：根据活动信息自动生成活动的宣传海报页面，宣传页面用于转发报名，页面包括该活动的基本信息和报名页面的二维码，扫码即可进入报名页面报名。可选：生成时自动附加时效性（如仅在报名时段有效）
    4. 活动信息在任何状态均可编辑、删除，编辑后可重新生成宣传页面，但保留已报名人员。
2. 报名
    1. 任何人可根据分享的链接或者扫码打开某个活动的报名页面，填写个人信息（姓名、手机号、学院）进行报名，报名后显示报名成功或失败。
    2. 报名需要验证以下条件：活动是否失效、个人填写信息数据验证、同一人不可重复报名、达到人数上限等。
3. 签到（可选）
    1. 基本签到：参与人可在会场中通过扫二维码，并输入手机号签到该活动。
    2. 准确签到（加分）：扫码输手机号签到，并且采用手机定位或者动态二维码（半分钟刷新）的方式防止虚假签到。（可选）使用国产数据库（如达梦数据库）

## 项目简介

本系统是高校学生活动管理的全流程解决方案，提供活动创建、报名管理、签到确认等核心功能，帮助学校高效组织和管理各类学生活动。

该项目实现了上述要求的所有功能，除了半分钟刷新二维码，使用国产数据库这两个要求。并在此基础上，实现了组合常用组件，如 Redis 和 RabbitMQ，实现了特定条件下的高可用。

## 注意事项

- 项目使用Java 21版本开发，请确保运行环境兼容
- 需要配置阿里云OSS服务用于文件存储
- 需要正确配置 Redis 和 RabbitMQ 服务地址
- 系统使用随机token进行身份验证，请妥善保管密钥信息
